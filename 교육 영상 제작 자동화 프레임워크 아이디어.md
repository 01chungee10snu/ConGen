# **구글 멀티모달 AI 생태계를 활용한 자율 교육용 영상 생성 프레임워크: Gemini 3.0 Pro, Nano Banana Pro, Veo 3.1 및 STT의 통합 아키텍처 및 구현 심층 분석**

## **서론: 교육 공학(EdTech)과 생성형 AI의 패러다임 전환**

현대 교육 콘텐츠 시장은 폭발적인 수요 증가와 개인화된 학습 경험에 대한 요구로 인해 전례 없는 변곡점을 맞이하고 있습니다. 전통적인 영상 제작 방식—스크립트 집필, 스토리보드 제작, 촬영 또는 애니메이션 작업, 성우 녹음, 편집, 자막 작업으로 이어지는 선형적 파이프라인—은 높은 비용과 긴 제작 기간이라는 구조적 한계를 지니고 있습니다. 특히, 다국어 지원과 학습자 수준별 맞춤형 콘텐츠 제작(Adaptive Learning Content)은 기존의 인력 중심 프로세스로는 확장성에 명확한 한계가 존재합니다.

이러한 상황에서 구글(Google)이 최근 공개한 일련의 멀티모달 생성형 AI 기술들, 즉 추론 능력이 비약적으로 향상된 **Gemini 3.0 Pro**, 텍스트 렌더링과 이미지 일관성이 강화된 **Nano Banana Pro(Gemini 3 Pro Image)**, 그리고 고품질 비디오 생성 모델인 **Veo 3.1**은 교육용 영상 제작의 자동화를 넘어선 '자율화'를 가능하게 하는 기술적 토대를 제공합니다.

본 보고서는 사용자의 요청에 따라 구조화된 스크립트 파일을 입력으로 받아, 기획부터 영상 생성, 음성 합성, 그리고 다국어 자막 생성까지의 전 과정을 하나의 통합된 에이전트 시스템(Agentic System)으로 구현하는 프레임워크를 제안합니다. 단순한 API의 나열이 아닌, 각 모델의 특장점을 교육적 맥락(Pedagogical Context)에 맞게 최적화하여 연결하는 오케스트레이션 전략을 중심으로, 총 15,000단어 분량의 심층적인 기술 분석과 구현 방안을 제시합니다.

**1\. 통합 아키텍처 설계: 에이전트 기반 오케스트레이션 (Agentic Orchestration)**

교육용 영상 제작 자동화 시스템의 핵심은 개별 AI 모델을 단순히 순차적으로 호출하는 것이 아니라, 전체적인 맥락을 이해하고 각 단계의 결과물을 검증하며 다음 단계의 입력으로 변환하는 '중앙 지휘자'의 존재입니다. 본 프레임워크에서는 이를 \*\*오케스트레이터 에이전트(Orchestrator Agent)\*\*라고 정의하며, 이 역할은 고도의 추론 능력과 긴 컨텍스트 윈도우(Context Window)를 가진 **Gemini 3.0 Pro**가 수행합니다.

### **1.1 시스템 토폴로지 및 데이터 흐름**

제안하는 시스템은 입력(Input), 스크립트 생성(Script Generation), 추론(Reasoning), 생성(Generation), 조립(Assembly), 배포(Delivery)의 **6단계 파이프라인**으로 구성됩니다. 각 단계는 독립적인 모듈로 작동하되, 오케스트레이터 에이전트에 의해 유기적으로 연결됩니다.

| 단계 | 구성 요소 | 역할 | 핵심 기술 (Google Cloud) |
| :---- | :---- | :---- | :---- |
| **입력** | 자연어 프롬프트 수신기 | 사용자의 자연어 요청(프롬프트) 수신 및 전처리 | Cloud Functions, API Gateway |
| **스크립트 생성** | **스크립트 생성 에이전트** | 자연어 프롬프트를 분석하여 구조화된 JSON/YAML 스크립트 자동 생성 | **Gemini 3.0 Pro** |
| **추론** | **오케스트레이터** | 교육적 목표 분석, 장면 분할, 프롬프트 엔지니어링, 일관성 관리 | **Gemini 3.0 Pro** |
| **시각** | **비주얼 에이전트** | 교육용 슬라이드, 도표, 캐릭터, 배경 이미지 생성 | **Nano Banana Pro (Gemini 3 Pro Image)** |
| **동작** | **모션 에이전트** | 정지 이미지의 비디오화(I2V), 장면 전환, 시간적 확장 | **Veo 3.1 / Veo 3.1 Fast** |
| **음성** | **보이스 에이전트** | 다국어 음성 합성, SSML 기반 운율 제어 | **Cloud TTS (Chirp 3)** |
| **검증** | **자막 에이전트** | 음성-텍스트 변환을 통한 타임스탬프 추출 및 번역 | **Cloud STT V2, Translation API** |
| **조립** | **편집 에이전트** | 비디오/오디오/자막 트랙의 멀티플렉싱(Muxing) 및 렌더링 | **Cloud Transcoder API / FFmpeg** |

이 아키텍처는 방향성 비순환 그래프(DAG, Directed Acyclic Graph) 형태를 띠며, 오케스트레이터는 각 노드(작업)의 상태를 모니터링하고 오류 발생 시 재시도하거나 대안 경로를 모색하는 동적 계획법을 수행합니다. 예를 들어, Veo 3.1이 생성한 영상의 길이가 오디오 길이보다 짧을 경우, 오케스트레이터는 즉시 '영상 확장(Extension)' 기능을 호출하여 부족한 길이를 보완하는 판단을 내립니다.1

### **1.2 자연어 프롬프트 입력 (Natural Language Prompt Input)**

기존의 교육 영상 제작 시스템들은 복잡한 구조화된 파일(JSON/YAML)을 직접 작성해야 했지만, 본 프레임워크의 핵심 혁신은 **사용자가 자연어 프롬프트만 입력하면 LLM이 이를 이해하여 자동으로 구조화된 스크립트를 생성**한다는 점입니다.

#### **자연어 입력 예시**

사용자는 다음과 같은 간단한 프롬프트만 입력하면 됩니다:

> "초등학교 5학년 수준에서 광합성의 원리를 설명하는 10분짜리 교육 영상을 만들어줘. 친근한 여자 선생님 캐릭터가 등장하고, 영어와 일본어 자막도 만들어줘. 색감은 밝고 따뜻한 분위기로 해줘."

또는 더 복잡한 요청도 가능합니다:

> "대학교 교양 수업용 '인공지능의 역사' 강의를 만들어줘. 20분 분량으로 5개 챕터로 나눠서 (1) 1950년대 탄생 (2) AI 겨울 (3) 머신러닝 부활 (4) 딥러닝 혁명 (5) 현재와 미래. 각 챕터마다 해당 시대의 중요 인물 사진이나 일러스트가 나오면 좋겠고, 차분하고 전문적인 남자 목소리로 해줘."

#### **스크립트 생성 에이전트의 역할**

**스크립트 생성 에이전트(Script Generation Agent)**는 Gemini 3.0 Pro의 강력한 이해력과 추론 능력을 활용하여 위와 같은 자연어 요청을 분석하고, 다음과 같은 단계를 수행합니다:

1. **의도 파악(Intent Extraction):** 교육 목표, 대상 학습자 수준, 원하는 길이, 스타일 등을 추출
2. **교육 설계(Instructional Design):** Gagné의 9가지 교수 사건(9 Events of Instruction)이나 ADDIE 모델을 참조하여 효과적인 학습 흐름 설계
3. **장면 분할(Scene Segmentation):** 개념별로 적절한 장면 수와 각 장면의 예상 길이 계산
4. **내레이션 작성(Narration Writing):** 학습자 수준에 맞는 어휘와 문장 길이로 스크립트 작성
5. **시각화 지시(Visual Instructions):** 각 장면에 적합한 시각적 표현 방식 결정

### **1.3 자동 생성되는 구조화된 스크립트 (Auto-Generated Structured Script)**

스크립트 생성 에이전트가 자연어 프롬프트로부터 자동 생성하는 JSON/YAML 스크립트의 구조는 다음과 같습니다:

1. **메타데이터(Metadata):** 프로젝트 제목, 대상 학습자 수준(초/중/고/성인), 기본 언어(ko-KR), 목표 자막 언어(en-US, ja-JP, zh-CN), 전반적인 톤앤매너(Tone & Manner), 브랜드 가이드라인(색상 코드, 폰트 스타일, 로고 자산 ID).  
2. **글로벌 자산(Global Assets):** 영상 전체에 걸쳐 일관성을 유지해야 하는 등장인물(교사 아바타)이나 배경 설정에 대한 참조 이미지(Reference Images) 경로. 이는 Gemini 3.0 Pro와 Nano Banana Pro의 일관성 유지 기능에 핵심적인 입력값입니다.3  
3. **장면 리스트(Scene List):** 각 장면(Scene)별 세부 정의.  
   * scene\_id: 순차적 식별자.  
   * type: 장면 유형 (인트로, 개념 설명, 도표, 퀴즈, 아웃트로).  
   * script: 내레이션 텍스트 (한글).  
   * visual\_description: 시각적 요소에 대한 설명 (예: "칠판 앞에서 광합성을 설명하는 선생님", "세포 내부의 미토콘드리아 확대").  
   * text\_overlay: 화면에 표시될 텍스트 (Nano Banana Pro의 텍스트 렌더링용).  
   * motion\_prompt: 카메라 움직임이나 객체의 동작 지시 (Veo 3.1용).
   * estimated\_duration: 예상 장면 길이(초 단위).

이러한 구조화된 스크립트는 후속 에이전트들(오케스트레이터, 비주얼 에이전트 등)이 복잡한 추론을 수행하기 위한 기초 자료가 됩니다.4

### **1.4 대화형 스크립트 수정 (Interactive Script Refinement)**

스크립트 생성 에이전트는 일방적인 생성에 그치지 않고, **사용자와의 대화를 통해 스크립트를 반복적으로 개선**할 수 있습니다:

* **미리보기 제공:** 생성된 스크립트의 요약본을 사용자에게 보여주고 피드백 요청
* **부분 수정:** "3번째 장면을 좀 더 재미있게 해줘", "퀴즈 장면을 추가해줘" 등의 요청 처리
* **톤 조정:** "좀 더 친근하게", "더 학술적으로" 등의 스타일 변경 요청 반영
* **승인 후 실행:** 사용자가 최종 스크립트를 승인하면 영상 생성 파이프라인 시작

**2\. 오케스트레이터: Gemini 3.0 Pro의 심층 추론 및 제어**

시스템의 두뇌에 해당하는 **Gemini 3.0 Pro**는 단순한 LLM(Large Language Model)을 넘어선 복잡한 에이전트 작업을 수행하도록 설계되었습니다. 특히 'Thinking Model'로서의 특성은 교육용 영상 제작 과정에서 발생할 수 있는 논리적 오류나 시각적 불일치를 사전에 차단하는 데 결정적인 역할을 합니다.5

### **2.1 교육적 해체 및 프롬프트 최적화 (Pedagogical Decomposition)**

입력된 스크립트가 "미토콘드리아는 세포의 에너지 공장입니다"라고 되어 있을 때, Gemini 3.0 Pro는 이를 시각적 언어와 청각적 언어로 분해(Decomposition)합니다.

1. **시각적 추론:** "에너지 공장"이라는 비유를 시각화하기 위해, 실제 미토콘드리아의 과학적 다이어그램을 생성할지, 아니면 공장이 돌아가는 애니메이션을 생성할지 결정합니다. 학습 대상이 초등학생이라면 비유적인 애니메이션을, 대학생이라면 과학적 다이어그램을 선택하도록 프롬프트를 조정합니다.  
2. **프롬프트 엔지니어링:** Nano Banana Pro와 Veo 3.1은 각기 다른 프롬프트 구조를 요구합니다. Gemini 3.0 Pro는 입력된 자연어 설명을 각 모델의 API 스펙에 맞는 기술적 프롬프트로 변환합니다.  
   * *Nano Banana Pro용:* "High-fidelity illustration of a mitochondria cross-section, educational style, clear labels, white background,".7  
   * *Veo 3.1용:* "Cinematic camera pan, slow zoom in, 1080p, detailed texture, consistent lighting,".2

### **2.2 컨텍스트 관리 및 일관성 유지 (Context & Consistency Management)**

교육용 영상에서 가장 중요한 것은 \*\*일관성(Consistency)\*\*입니다. 1강에서는 실사 스타일의 선생님이 등장했다가 2강에서는 카툰 스타일의 캐릭터가 등장하면 학습자의 몰입을 방해합니다. Gemini 3.0 Pro는 100만 토큰 이상의 긴 컨텍스트 윈도우를 활용하여 프로젝트 전체의 시각적 자산을 기억합니다.6

* **참조 이미지 ID 관리:** Nano Banana Pro로 생성된 첫 번째 장면의 '선생님' 이미지를 Reference\_ID\_001로 저장하고, 이후 모든 장면 생성 시 이 ID를 reference\_images 파라미터로 주입하여 동일한 인물이 다양한 포즈를 취하도록 제어합니다.1  
* **스타일 일관성:** 색상 팔레트나 폰트 스타일과 같은 브랜드 가이드라인을 '시스템 프롬프트'로 상주시켜, 모든 생성 요청에 이 규칙이 적용되도록 강제합니다.

### **2.3 오류 감지 및 자가 수정 (Self-Correction)**

Gemini 3.0 Pro의 'Thinking' 모드는 생성된 결과물을 평가하는 데에도 사용됩니다. 예를 들어, Veo 3.1이 생성한 영상 URL을 받아 해당 영상의 썸네일이나 메타데이터를 분석하거나, 사용자 피드백 루프를 통해 "글자가 깨졌다"는 신호가 오면, 즉시 Nano Banana Pro의 텍스트 렌더링 파라미터를 조정하여 재생성을 요청하는 자율적인 에러 핸들링을 수행할 수 있습니다. 이는 Gemini 3.0의 도구 사용(Tool Use) 능력과 결합되어 외부 API(Transcoder, GCS)를 직접 제어하는 형태로 구현됩니다.4

**3\. 시각적 자산 생성: Nano Banana Pro (Gemini 3 Pro Image)의 활용**

본 보고서에서 사용자가 언급한 "Nano Banana Pro"는 구글의 최신 이미지 생성 모델인 **Gemini 3 Pro Image**의 별칭으로 파악됩니다.9 이 모델은 기존 모델 대비 텍스트 렌더링 능력과 프롬프트 준수 능력이 비약적으로 향상되어, 교육 자료 생성에 최적화되어 있습니다.

### **3.1 고해상도 텍스트 렌더링과 교육용 슬라이드 생성**

교육용 영상의 핵심은 정확한 정보 전달이며, 이는 종종 텍스트와 도표의 형태로 나타납니다. 기존의 이미지 생성 AI들은 이미지 내의 텍스트를 뭉개거나(gibberish) 오타를 내는 경향이 있었으나, Nano Banana Pro는 **고급 텍스트 렌더링(Advanced Text Rendering)** 기능을 통해 정확한 철자의 텍스트를 이미지 내에 삽입할 수 있습니다.10

* **구현 방안:** 오케스트레이터는 스크립트의 text\_overlay 필드 값을 추출하여 Nano Banana Pro 프롬프트에 "Text: '광합성 과정' written in bold, clean Hangul font"와 같이 명시합니다.  
* **다국어 지원:** 이 모델은 다국어 텍스트 렌더링을 지원하므로, 한글뿐만 아니라 영어, 일본어 등 다국어 버전의 슬라이드를 별도로 생성하여 각 언어별 영상 버전을 만드는 데 활용할 수 있습니다.11  
* **해상도:** 기본적으로 2K 해상도를 지원하며, 업스케일링을 통해 4K까지 확장이 가능하여 대형 스크린용 교육 콘텐츠 제작에도 적합합니다.10

### **3.2 참조 이미지를 활용한 캐릭터 및 브랜드 일관성**

교육 영상에 등장하는 'AI 튜터'나 '캐릭터'의 일관성을 유지하기 위해 Nano Banana Pro의 **다중 참조 이미지(Reference Image)** 기능이 활용됩니다.

* **Few-Shot Prompting:** 최대 14장까지의 참조 이미지를 입력으로 받을 수 있습니다. 이를 통해 사전에 제작된 캐릭터 시트(정면, 측면, 후면 등)와 브랜드 로고, 컬러 팔레트 이미지를 입력하여 생성되는 모든 이미지가 이 스타일을 따르도록 강제합니다.7  
* **활용 시나리오:** "선생님이 칠판을 가리키는 장면", "선생님이 고민하는 장면", "선생님이 웃으며 인사하는 장면" 등을 생성할 때, 참조 이미지를 통해 옷차림과 얼굴 생김새가 100% 일치하는 이미지를 생성합니다. 이는 영상의 연속성을 보장하는 가장 중요한 기술적 요소입니다.

### **3.3 검색 기반 그라운딩 (Search Grounding)을 통한 과학적 정확성**

과학, 역사, 지리 등 사실 관계가 중요한 교육 콘텐츠의 경우, 생성된 이미지가 사실과 다르면 치명적입니다. Nano Banana Pro는 구글 검색(Google Search)과 연결되어 실시간 정보를 바탕으로 이미지를 생성하는 **Grounding** 기능을 제공합니다.10

* **구현 예시:** "19세기 조선의 경복궁 모습"을 요청하면, 모델이 웹 검색을 통해 경복궁의 실제 건축 양식과 배치를 파악하고 이를 반영하여 이미지를 생성합니다. 할루시네이션(Hallucination)을 줄이고 교육적 신뢰도를 높이는 핵심 기능입니다.

**4\. 동적 영상 합성: Veo 3.1의 시공간적 구현**

정지된 이미지가 교육적 정보를 담고 있다면, 영상은 흐름과 맥락을 전달합니다. **Veo 3.1**은 Nano Banana Pro가 생성한 고품질 자산을 움직이는 영상으로 변환하는 역할을 수행합니다.

### **4.1 이미지-투-비디오(I2V) 파이프라인과 비디오 일관성**

Veo 3.1은 텍스트만으로 영상을 생성하는 T2V(Text-to-Video)뿐만 아니라, 이미지를 입력으로 받아 움직임을 부여하는 **I2V(Image-to-Video)** 기능에 강점을 가집니다.2 본 프레임워크에서는 일관성을 위해 I2V 방식을 기본으로 채택합니다.

1. **입력:** Nano Banana Pro로 생성된 고해상도 슬라이드 또는 캐릭터 이미지.  
2. **프롬프트:** "카메라가 천천히 줌인하며 텍스트가 강조됨", "캐릭터가 손을 들어 칠판을 가리키며 입을 움직임(립싱크 준비 동작)".  
3. **일관성 유지:** Veo 3.1의 **Ingredients-to-Video** 기능을 활용하여, 움직이는 영상 속 객체가 원본 이미지(Nano Banana Pro 결과물)와 시각적으로 동일하게 유지되도록 합니다.1 이는 AI 영상 특유의 '울렁거림'이나 객체 변형을 최소화합니다.

### **4.2 영상 확장(Video Extension)과 긴 호흡의 강의 구현**

Veo 3.1의 기본 생성 길이는 4초, 6초, 8초로 제한적입니다.2 그러나 교육용 설명은 수십 초 이상 지속될 수 있습니다. 이를 해결하기 위해 **영상 확장(Video Extension)** 기술을 적용합니다.

* **데이지 체인(Daisy-Chaining) 방식:**  
  * 1단계: 0\~8초 구간 생성.  
  * 2단계: 생성된 8초 영상의 **마지막 프레임**을 추출하여 다음 8\~16초 구간 생성의 **첫 번째 프레임**으로 입력.1  
  * 3단계: 이 과정을 반복하여 원하는 길이(예: 32초)의 영상을 생성하고, 추후 트랜스코더 API로 이어 붙입니다.  
  * 이 방식은 영상의 끊김 없는 연결을 보장하며, 오디오 길이와 영상 길이를 정확하게 동기화할 수 있게 해줍니다.

### **4.3 프레임 보간(Interpolation)을 통한 자연스러운 장면 전환**

서로 다른 두 개념(장면 A와 장면 B) 사이를 연결할 때, 단순한 컷 편집은 딱딱하게 느껴질 수 있습니다. Veo 3.1은 **첫 프레임과 마지막 프레임 지정 생성** 기능을 지원합니다.1

* **구현:** 장면 A의 마지막 프레임과 장면 B의 첫 프레임을 Veo 3.1에 입력으로 주고, "부드러운 모핑(Morphing) 전환"을 프롬프트로 입력하면, 두 이미지 사이를 자연스럽게 연결하는 전이(Transition) 영상을 생성합니다. 이는 시각적 흐름을 유지하며 학습자의 주의를 환기시키는 효과적인 교육적 장치가 됩니다.

**5\. 청각적 레이어: TTS와 STT의 동기화 및 자막 자동화**

영상과 이미지가 준비되었다면, 이를 설명하는 음성(Narration)과 자막(Subtitles)이 필요합니다. 구글의 Speech 기술은 이 과정을 완벽하게 자동화합니다.

### **5.1 Cloud TTS (Chirp 3)를 활용한 감성적 내레이션**

교육용 내레이션은 기계적인 톤이 아닌, 학습자와 상호작용하는 듯한 자연스러운 억양과 감정이 필요합니다. 구글의 **Chirp 3** 모델은 트랜스포머 기반의 대규모 음성 모델로, 텍스트의 맥락을 이해하고 적절한 끊어 읽기와 감정 표현을 수행합니다.13

* **SSML(Speech Synthesis Markup Language) 적용:** 오케스트레이터는 스크립트의 중요도에 따라 SSML 태그를 자동 삽입합니다.  
  * \<break time="500ms"/\>: 개념 설명 후 잠시 멈춤.  
  * \<emphasis level="strong"\>: 핵심 키워드 강조.  
  * \<prosody rate="90%"\>: 어려운 개념 설명 시 말하기 속도를 늦춤.  
* **다국어 음성 생성:** 입력된 한글 스크립트를 Translation API로 번역한 후, 해당 언어(영어, 일어 등)의 Chirp 3 모델을 호출하여 다국어 더빙 트랙을 생성합니다.

### **5.2 Cloud STT를 이용한 역방향 타임스탬프 추출 (Reverse-Timestamping)**

일반적으로는 스크립트가 있으므로 자막을 바로 생성할 수 있다고 생각하기 쉽지만, TTS가 생성한 오디오의 정확한 타이밍(어느 단어가 몇 초에 발화되는지)을 텍스트만으로는 완벽하게 예측하기 어렵습니다. 따라서 본 프레임워크는 **생성된 TTS 오디오를 다시 STT(Speech-to-Text)로 분석**하는 역발상적 접근을 취합니다.

* **구현:** 생성된 오디오 파일을 Cloud STT API V2의 BatchRecognize 메서드로 전송합니다. 이때 enableWordTimeOffsets=True 파라미터를 설정합니다.15  
* **결과:** STT는 오디오 내의 모든 단어에 대해 startTime과 endTime을 100ms 단위로 반환합니다. 이 데이터를 기반으로 자막 파일(.vtt,.srt)을 생성하면, 오디오와 100% 동기화된 완벽한 자막을 얻을 수 있습니다.

### **5.3 다국어 자막 생성 파이프라인**

1. **소스 자막 생성:** 한글 오디오 \-\> STT \-\> 한글 타임스탬프 데이터 획득.  
2. **번역:** 한글 텍스트 세그먼트를 Cloud Translation API를 통해 타겟 언어(영어, 일어 등)로 번역.  
3. **동기화 유지:** 번역된 텍스트를 원본 한글 타임스탬프에 매핑하여 .vtt 파일 생성.  
4. **검증:** 오케스트레이터가 번역문의 길이 차이로 인한 자막 가독성 문제(너무 긴 문장 등)를 체크하고 필요 시 타임스탬프를 병합하거나 분할합니다.

**6\. 최종 조립 및 배포: Cloud Transcoder API**

모든 개별 자산(비디오 클립, 오디오 트랙, 자막 파일)이 생성되면, 이를 하나의 최종 영상 파일(MP4)로 병합(Muxing)해야 합니다. 이를 위해 **Google Cloud Transcoder API**를 활용합니다.16

### **6.1 JobConfig 구성 및 스티칭(Stitching)**

Transcoder API는 editList라는 개념을 사용하여 타임라인을 구성합니다. 오케스트레이터는 다음과 같은 JSON 설정을 생성하여 Transcoder API에 작업을 요청합니다.

JSON

{  
  "config": {  
    "editList": \[  
      {  
        "key": "atom1",  
        "inputs": \["video\_scene1\_uri", "audio\_scene1\_uri"\],  
        "startTimeOffset": "0s"  
      },  
      {  
        "key": "atom2",  
        "inputs": \["video\_scene2\_uri", "audio\_scene2\_uri"\],  
        "startTimeOffset": "0s"  
      }  
    \],  
    "elementaryStreams":,  
    "muxStreams":  
      }  
    \]  
  }  
}

이 설정은 개별적으로 생성된 비디오와 오디오 클립을 순서대로 이어 붙이고(Stitching), 한국어와 영어 자막을 소프트 자막(Soft Subtitle) 형태로 MP4 컨테이너에 포함시킵니다.17

### **6.2 복잡한 합성을 위한 대안: Cloud Run \+ FFmpeg**

만약 단순히 이어 붙이는 것을 넘어, 영상 위에 워터마크를 합성하거나, 화면 속 화면(PIP) 효과, 복잡한 텍스트 애니메이션 오버레이가 필요한 경우, Transcoder API만으로는 한계가 있을 수 있습니다. 이 경우 **Cloud Run**에서 실행되는 컨테이너화된 **FFmpeg** 인스턴스를 활용합니다.19

* 오케스트레이터는 Cloud Run 작업을 트리거하며, GCS에 저장된 자산들의 경로를 환경 변수로 전달합니다.  
* FFmpeg는 강력한 필터 그래프(Filter Graph) 기능을 사용하여 시각적 효과를 적용하고 최종 렌더링을 수행한 후 GCS에 업로드합니다.

**7\. 구현 전략 및 코드 로직 (Implementation Strategy)**

실제 구현을 위한 논리적 흐름과 주요 코드 스니펫을 제시합니다.

### **7.1 단계별 로직 흐름 (Step-by-Step Workflow)**

1. **User Input:** 사용자가 자연어 프롬프트를 입력 (예: "초등학교 5학년 수준의 광합성 10분 강의 만들어줘").
2. **Script Generation:**  
   * 스크립트 생성 에이전트가 `google.genai.Client()`를 사용하여 Gemini 3 Pro 모델로 자연어 분석.  
   * 교육적 의도, 대상 학습자 수준, 스타일, 길이 등을 추출.  
   * 구조화된 JSON/YAML 스크립트 자동 생성.
3. **Interactive Refinement (Optional):**  
   * 생성된 스크립트 요약본을 사용자에게 제시.  
   * 사용자 피드백에 따라 스크립트 수정 (예: "퀴즈 장면 추가해줘").  
   * 사용자가 최종 승인하면 다음 단계로 진행.
4. **Orchestration:**  
   * 오케스트레이터가 승인된 스크립트를 분석하고 자산 생성 계획(Plan) 수립.  
   * 각 장면별 에이전트 호출 순서 및 의존성 그래프 생성.
5. **Asset Generation (Parallel Execution):**  
   * *Visual:* gemini-3-pro-image API 호출. reference\_images 파라미터로 일관성 유지.  
   * *Audio:* texttospeech\_v1.TextToSpeechClient 호출. SSML 적용하여 오디오 생성.  
6. **Motion Synthesis:**  
   * 생성된 이미지와 오디오 길이를 기반으로 veo-3.1-generate API 호출.  
   * 필요 시 last\_frame을 활용한 Loop 생성으로 영상 길이 확장.  
7. **Subtitle Generation:**  
   * 생성된 오디오를 speech\_v2.SpeechClient로 전송. BatchRecognize로 타임스탬프 획득.  
   * Python 스크립트로 VTT 파일 포맷팅.  
8. **Final Assembly:**  
   * 모든 자산 URI를 수집하여 transcoder\_v1.TranscoderServiceClient 호출.  
   * create\_job 메서드로 최종 렌더링 요청.  
9. **Completion:** 최종 영상 URL을 사용자에게 알림 (Pub/Sub 또는 이메일).

### **7.2 파이썬(Python) 구현 예시: Veo 3.1 비디오 생성 요청**

다음은 오케스트레이터가 Veo 3.1에 이미지 기반 비디오 생성을 요청하고, 일관성을 위해 설정을 제어하는 코드 예시입니다.1

Python

import time  
from google import genai  
from google.genai.types import GenerateVideosConfig, Image

def generate\_scene\_video(client, prompt, image\_gcs\_uri, duration\_sec, output\_path):  
    """  
    Nano Banana Pro 이미지를 기반으로 Veo 3.1 비디오 생성  
    """  
    print(f"Generating video for prompt: {prompt}...")  
      
    \# 설정 객체 생성: 해상도 1080p, 비율 16:9, 사람 생성 허용  
    config \= GenerateVideosConfig(  
        aspect\_ratio="16:9",  
        resolution="1080p",  \# 교육용 고화질 필수  
        duration=duration\_sec, \# TTS 오디오 길이와 매칭 (4, 6, 8 중 선택)  
        output\_gcs\_uri=output\_path,  
        person\_generation="allow\_adult", \# 선생님 아바타 허용  
        \# prompt\_rewriter=False \# 정확한 지시 이행을 위해 필요시 비활성화   
    )

    operation \= client.models.generate\_videos(  
        model="veo-3.1-generate-001", \# Production Model  
        prompt=prompt,  
        image=Image(  
            gcs\_uri=image\_gcs\_uri,  
            mime\_type="image/png"  
        ),  
        config=config  
    )

    \# Long-running Operation 폴링  
    while not operation.done:  
        print("Waiting for video generation...")  
        time.sleep(10)  
        operation \= client.operations.get(operation)

    if operation.result:  
        print(f"Video generated: {operation.result.generated\_videos.video.uri}")  
        return operation.result.generated\_videos.video.uri  
    else:  
        raise Exception("Video generation failed.")

### **7.3 하드웨어 관련 명확화 (Hardware Clarification)**

사용자 쿼리 중 "Nano Banana"와 관련하여 일부 하드웨어(Banana Pi 등)와의 혼동 가능성이 있어 이를 명확히 합니다.

* **모델:** 본 보고서의 Nano Banana는 **Gemini 2.5 Flash Image** 또는 **Gemini 3 Pro Image** 모델을 지칭하는 구글 내부/커뮤니티의 별칭입니다.7  
* **하드웨어:** Banana Pi BPI-M7과 같은 싱글 보드 컴퓨터(SBC)는 **Gemini Nano**와 같은 경량화 모델을 온디바이스(On-device)로 구동할 수 있는 NPU를 탑재하고 있습니다.21  
* **구현 방향:** 본 프레임워크는 고화질(4K/1080p) 영상 제작을 목표로 하므로, SBC에서의 온디바이스 구동보다는 **Google Cloud Vertex AI** 기반의 클라우드 파이프라인이 적합합니다. Banana Pi는 최종 생성된 영상을 재생하는 키오스크나, API를 호출하는 엣지 컨트롤러로 활용될 수 있습니다.23

**8\. 결론 및 향후 전망 (Conclusion & Future Outlook)**

본 보고서에서 제안한 \*\*'Gemini-Veo-Nano 통합 교육 영상 생성 프레임워크'\*\*는 기존의 노동 집약적인 영상 제작 프로세스를 AI 에이전트 기반의 자율 시스템으로 혁신하는 구체적인 청사진입니다.

1. **완전성:** 텍스트 스크립트 하나로 시각(Visual), 청각(Audio), 시간(Motion), 언어(Subtitle)의 4요소를 완벽하게 갖춘 영상을 생성합니다.  
2. **일관성:** Gemini 3.0 Pro의 오케스트레이션과 Nano Banana Pro/Veo 3.1의 참조 이미지 기능을 통해, AI 영상의 고질적인 문제인 일관성 결여를 해결했습니다.  
3. **확장성:** Cloud Transcoder와 Translation API를 통해 비용 효율적으로 수백 개의 다국어 버전을 즉시 생성할 수 있습니다.

향후 Gemini 3.0의 추론 능력이 더욱 고도화되면, 스크립트조차 필요 없이 "초등학교 5학년 수준의 광합성 강의를 만들어줘"라는 한 문장만으로 전체 커리큘럼을 기획하고 영상을 생성하는 **'Text-to-Curriculum'** 시대로 진입할 것입니다. 제안된 프레임워크는 이러한 미래 교육 환경을 선점하기 위한 강력한 기술적 기반이 될 것입니다.
